## 📋 概述

## 🐛 故障排查

### 问题 1：播放无声音

**症状：** 显示"✅ 播放完成"，但耳机听不到声音

**可能原因：**
1. 耳机未插好
2. Playback 音量过低
3. Deviceid 未设置
4. 播放设备错误

**解决方案：**

```bash
# 1. 检查耳机连接
# 确保耳机插入 3.5mm 接口

# 2. 手动设置播放参数
amixer set Playback 10
amixer set Deviceid 2

# 3. 测试播放官方音频
aplay -Dhw:ascend310b /opt/opi_test/audio/tianlu.wav

# 4. 如果官方音频能播放，重新运行测试
python3 scripts/test_playback.py
```

### 问题 2：TTS 生成失败

**症状：** 提示"❌ 语音生成失败"

**可能原因：**
1. 网络连接问题（Edge-TTS 需要网络）
2. 输出目录不存在
3. 磁盘空间不足

**解决方案：**

```bash
# 1. 检查网络连接
ping -c 3 www.baidu.com

# 2. 检查输出目录
mkdir -p output/tts

# 3. 检查磁盘空间
df -h

# 4. 重新运行测试
python3 scripts/test_playback.py
```

### 问题 3：播放有杂音或失真

**症状：** 能听到声音，但音质差

**可能原因：**
1. Playback 音量过高
2. 音频文件损坏
3. 播放设备问题

**解决方案：**

```bash
# 1. 降低 Playback 音量
amixer set Playback 5

# 2. 检查音频文件
ffmpeg -i output/tts/tts_*.wav

# 3. 测试播放官方音频
aplay -Dhw:ascend310b /opt/opi_test/audio/tianlu.wav

# 4. 重新运行测试
python3 scripts/test_playback.py
```

### 问题 4：播放器卡住

**症状：** 显示"🔊 开始播放！"后卡住

**可能原因：**
1. 音频设备被占用
2. aplay 进程异常

**解决方案：**

```bash
# 1. 查找占用音频设备的进程
lsof /dev/snd/*

# 2. 杀死占用进程
kill -9 <PID>

# 3. 重新运行测试
python3 scripts/test_playback.py
```

## 🔬 技术细节

### 播放设备说明

**为什么使用 `-Dhw:ascend310b`？**

```python
aplay -Dhw:ascend310b audio_file
```

- Orange Pi AI Pro 的音频芯片是 Ascend 310B
- 必须指定播放设备才能正常输出到耳机
- 如果不指定，可能会使用错误的设备

### 音频参数说明

**Deviceid = 2：**
- 指定使用耳机/麦克风设备
- 2 = 3.5mm 接口

**Playback = 10：**
- 播放音量设置
- 范围：0-100
- 推荐值：5-15

### AudioPlayer 工作原理

**异步播放队列：**
```python
# 添加到队列
await player.play_async(audio_file)

# 播放工作线程从队列取出并播放
while self.is_running:
    audio_file = await self.play_queue.get()
    await self._play_file(audio_file)
```

**优点：**
- ✅ 支持队列播放
- ✅ 支持打断机制
- ✅ 异步非阻塞
