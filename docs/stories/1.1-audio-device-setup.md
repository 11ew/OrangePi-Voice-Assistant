## 🔧 技术细节

### 硬件平台
- **开发板:** Orange Pi AI Pro (Ascend 310B)
- **录音设备:** 板载麦克风或外接麦克风
- **播放设备:** 3.5mm 耳机接口

### 关键配置参数

#### 录音设备
```bash
# 设备名称
Device: plughw:0,1

# 录音参数
Format: S16_LE
Sample Rate: 16000 Hz
Channels: 1 (Mono)
```

#### 播放设备
```bash
# 设备名称
Device: hw:ascend310b (或 hw:0,0)

# 播放参数
Format: S16_LE
Sample Rate: 16000 Hz (或更高)
Channels: 1 (Mono)
```

#### 音量设置
```bash
# 录音音量
amixer set Capture 10

# 播放音量
amixer set Playback 10

# 设备ID（耳机）
amixer set Deviceid 2
```

### 依赖工具
- `alsa-utils` - ALSA 音频工具集
- `arecord` - 录音命令
- `aplay` - 播放命令
- `amixer` - 音量控制命令

## 🧪 测试要点

### 测试 1: 查看音频设备
```bash
# 查看录音设备
arecord -l

# 预期输出：
# card 0: ascend310b [ascend310b], device 1: ...

# 查看播放设备
aplay -l

# 预期输出：
# card 0: ascend310b [ascend310b], device 0: ...
```

### 测试 2: 录音测试
```bash
# 录音 5 秒
arecord -D plughw:0,1 -f S16_LE -r 16000 -c 1 -t wav -d 5 test_record.wav

# 检查文件大小（应该约 160KB）
ls -lh test_record.wav

# 检查文件信息
file test_record.wav
```

### 测试 3: 播放测试
```bash
# 播放录音文件
aplay -Dhw:ascend310b test_record.wav

# 预期结果：
# - 能听到自己的声音
# - 声音清晰无杂音
# - 音量适中
```

### 测试 4: 音量调整测试
```bash
# 设置录音音量
amixer set Capture 10

# 设置播放音量
amixer set Playback 10

# 设置设备ID
amixer set Deviceid 2

# 重新测试录音和播放
```

### 测试 5: 长时间录音测试
```bash

arecord -D plughw:0,1 -f S16_LE -r 16000 -c 1 -t wav -d 30 test_long.wav

```

## ⚠️ 常见问题

### 问题 1: 录音无声音或全是噪音
**原因：**
- 麦克风音量太低或太高
- 设备名称不正确
- 麦克风未正确连接

**解决方案：**
1. 调整录音音量：`amixer set Capture 10`
2. 确认使用 `plughw:0,1` 而不是 `hw:0,1`
3. 检查麦克风连接

### 问题 2: 播放无声音
**原因：**
- 耳机未插入
- 播放音量为 0
- 设备ID未设置为耳机

**解决方案：**
1. 确认耳机已插入 3.5mm 接口
2. 调整播放音量：`amixer set Playback 10`
3. 设置设备ID：`amixer set Deviceid 2`

### 问题 3: 设备忙碌（Device or resource busy）
**原因：**
- 其他进程正在使用音频设备

**解决方案：**
1. 查找占用进程：`lsof /dev/snd/*`
2. 杀死占用进程：`kill -9 <PID>`
3. 或重启 ALSA：`sudo systemctl restart alsa-restore`

### 问题 4: 采样率不支持
**原因：**
- 硬件不支持指定的采样率

**解决方案：**
- 使用 `plughw:0,1` 而不是 `hw:0,1`（plughw 支持自动转换）
