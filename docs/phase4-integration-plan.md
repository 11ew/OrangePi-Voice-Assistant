### 任务 4.3：端到端集成测试

**目标：** 运行完整的实时语音助手系统

**测试步骤：**

#### 步骤 1：首次启动测试


**预期行为：**
1. ✅ 所有模块初始化成功
2. ✅ 开始监听音频输入
3. ✅ 状态机转换到"监听中"

#### 步骤 2：单轮对话测试

**测试流程：**
```
用户说话 → VAD 检测 → ASR 识别 → LLM 回复 → TTS 合成 → 播放
```

**测试用例：**

1. **简单问候**
   - 输入：说"你好"
   - 预期：识别正确，回复友好，播放清晰

2. **中文问题**
   - 输入：说"今天天气怎么样？"
   - 预期：识别正确，回复合理，播放清晰

3. **英文问题**
   - 输入：说"What's your name?"
   - 预期：识别正确，回复合理，播放清晰

4. **中英混合**
   - 输入：说"Hello，你好"
   - 预期：识别正确，回复合理，播放清晰

**验收标准：**
- [ ] VAD 能检测到语音开始和结束
- [ ] ASR 识别准确率 > 90%
- [ ] LLM 回复合理且简洁
- [ ] TTS 语音清晰自然
- [ ] 总延迟 < 8 秒（首次可以放宽）

#### 步骤 3：多轮对话测试

**测试流程：**
连续进行 3-5 轮对话，测试系统稳定性

**测试用例：**
1. 第 1 轮：你好
2. 第 2 轮：你叫什么名字？
3. 第 3 轮：你能做什么？
4. 第 4 轮：谢谢
5. 第 5 轮：再见

**验收标准：**
- [ ] 所有轮次都能正常完成
- [ ] 状态转换正确
- [ ] 无内存泄漏
- [ ] 无资源冲突
- [ ] 错误恢复机制正常

#### 步骤 4：异常情况测试

**测试场景：**

1. **静音测试**
   - 场景：长时间静音

### 任务 4.4：问题修复和优化

**目标：** 修复集成测试中发现的问题

**常见问题和解决方案：**

#### 问题 1：音频设备冲突

**现象：** `Device or resource busy`

**解决方案：**
```bash
# 查找占用进程
lsof /dev/snd/*

# 杀死占用进程
kill -9 <PID>
```

#### 问题 2：模块导入失败

**现象：** `ModuleNotFoundError`

**解决方案：**
```bash
# 检查 Python 路径
echo $PYTHONPATH

# 安装缺失的包
pip install <package_name>
```

#### 问题 3：配置文件错误

**现象：** `KeyError` 或 `FileNotFoundError`

**解决方案：**
- 检查配置文件路径
- 验证 JSON 格式
- 补充缺失的配置项

#### 问题 4：性能问题

**现象：** 延迟过高，响应慢

**优化方向：**
1. 使用 INT8 量化模型（已使用）
2. 调整 VAD 参数减少误触发
3. 优化音频缓冲区大小
4. 使用异步并行处理（已实现）

#### 问题 5：识别准确率低

**现象：** ASR 识别错误多

**优化方向：**
1. 调整录音音量（已知音量反向特性）
2. 优化 VAD 参数
3. 考虑添加降噪（RNNoise 已验证不推荐）
4. 更换更好的麦克风
