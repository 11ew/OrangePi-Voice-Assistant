## 🎯 问题背景

用户在测试实时语音助手系统时，发现三个严重问题：

1. **语音识别准确率极低** - 用户说话内容与识别结果相差很大
2. **播放无声音** - 播放器显示完成，但耳机听不到声音
3. **录音进程崩溃** - arecord 进程意外结束

---

## 🔍 问题诊断

### 问题 1：语音识别准确率极低

**用户反馈：**
```
用户说："你好，我是哈雷酱"
识别结果："嗯"
准确率：< 10%
```

**根本原因：**
- **采样率不匹配** - 硬件 48000 Hz vs 代码 16000 Hz
- 导致音频失真，产生"魔音"效果
- ASR 模型无法识别失真的音频

---

## 🛠️ 修复方案

### 核心思路

**先录制硬件原生采样率，再软件降采样**

```
录音（48000 Hz）→ 降采样（÷3）→ 输出（16000 Hz）
```

### 修复内容

#### 1. 修改 audio_capture.py

- 硬件采样率改为 48000 Hz
- 添加降采样逻辑（每 3 个样本取 1 个）
- 目标采样率保持 16000 Hz（ASR 需要）

#### 2. 创建测试脚本

**scripts/test_asr_recognition.py**
- 完整的 ASR 识别测试工具
- 录音 5 秒 + 自动识别
- 验证降采样逻辑

#### 3. 创建文档

- **docs/asr-test-script-guide.md** - 测试脚本使用指南
- **docs/audio-capture-sample-rate-fix.md** - 问题修复报告
