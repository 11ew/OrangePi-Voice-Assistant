## 🚀 V2 优化方案

### 优化 1: LLM 参数调整 ⭐

**目标：** 减少 LLM 生成时间

**实施内容：**

1. **降低 max_tokens**
   ```json
   "max_tokens": 100 → 50
   ```
   - 限制回复长度，加快生成速度
   - 预计提速：30-40%

2. **简化 system_prompt**
   ```
   原始: "你是一个俏皮可爱的语音助手，名叫哈雷酱。说话要像真人一样自然亲切，
         多用「呀」「哦」「呢」「啦」「嘛」等语气词，可以用「～」「！」表达情绪。
         回复要简短（1-2句话，30字以内），像朋友聊天一样轻松随意。"

   优化: "你是哈雷酱，俏皮可爱的语音助手。说话自然亲切，多用「呀」「哦」「呢」
         「啦」等语气词。回复超简短（1句话，15字内），像朋友聊天。"
   ```
   - 明确要求超简短回复（15字内）
   - 减少不必要的描述

**预计效果：**
- LLM 生成时间：~2秒 → ~1.2秒
- 提速：30-40%

### 优化 3: 更激进的句子分割 ⭐⭐⭐

**目标：** 更早开始 TTS 合成

**实施内容：**

```python
# 原始策略：只在遇到句子结束符时分割
if any(punct in sentence_buffer for punct in ['。', '！', '？', '.', '!', '?']):
    # 开始 TTS

# 优化策略：累积 8 个字就开始 TTS
should_split = (
    any(punct in sentence_buffer for punct in ['。', '！', '？', '.', '!', '?']) or
    word_count >= 8  # 累积 8 个字就分割
)
```

**优势：**
- 不等待完整句子，更早开始 TTS
- 降低首字延迟
- 提升流畅度

**预计效果：**
- 首字延迟：1-2秒 → 0.8-1.5秒
- 提速：20-30%

## ⚠️ 注意事项

### 1. 回复可能太简短

**现象：** 回复只有 10-15 个字

**调整方法：**
```json
"max_tokens": 50 → 80  // 稍微长一点
"system_prompt": "回复超简短（1句话，15字内）" → "回复简短（1-2句话，25字内）"
```

### 2. 语速可能太快

**现象：** 播放速度太快，听不清

**调整方法：**
```json
"rate": "+25%" → "+15%"  // 稍微慢一点
```

### 3. 短句可能太碎

**现象：** 8 个字就分割，句子太短

**调整方法：**
```python
word_count >= 8 → word_count >= 12  // 等更多字再分割
```
