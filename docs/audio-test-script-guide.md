## 📋 概述

本脚本是一个完整的自动化音频测试工具，用于验证 Orange Pi AI Pro 的录音和播放功能。

### 主要功能

1. **自动设置音频参数** - Capture、Deviceid、Playback
2. **录音测试** - 48000 Hz RAW 格式，避免 Orange Pi WAV bug
3. **格式转换** - 自动使用 ffmpeg 转换 PCM → WAV
4. **质量验证** - 自动检查音频信号强度
5. **播放测试** - 自动播放录音，验证完整流程

## 📊 测试流程

### 步骤 1：设置音频参数

脚本会自动设置以下参数：

```bash
amixer set Capture 10      # 录音音量（值越小信号越强）
amixer set Deviceid 2      # 设备 ID（2 = 耳机/麦克风）
amixer set Playback 10     # 播放音量
```

**输出示例：**
```
📋 步骤 1/5: 设置音频参数...
  设置录音音量 (Capture = 10)...
  设置设备 ID (Deviceid = 2)...
  设置播放音量 (Playback = 10)...
✅ 音频参数设置完成！
```

### 步骤 2：录音测试

脚本会倒计时 3 秒后开始录音 5 秒：

```bash
arecord -D plughw:0,1 -f S16_LE -r 48000 -c 1 -t raw -d 5 test_48k.pcm
```

**输出示例：**
```
============================================================
🎤 步骤 2/5: 录音测试（5 秒）
============================================================

⚠️  请对着麦克风清晰地说话...
   （录音将在 3 秒后开始）

3...
2...
1...

🔴 开始录音！（5 秒）

正在录音 原始資料 'test_48k.pcm' : Signed 16 bit Little Endian, 频率48000Hz， Mono

✅ 录音完成！
```

**注意事项：**
- 📢 看到倒计时后，准备对着麦克风说话
- 🎤 说话要清晰，音量适中
- ⏱️ 录音时长固定为 5 秒
- 🔇 在安静环境下测试效果最佳

### 步骤 3：格式转换

脚本会自动将 PCM 转换为 WAV 格式：

```bash
ffmpeg -f s16le -ar 48000 -ac 1 -i test_48k.pcm -y test_48k.wav
```

**输出示例：**
```
============================================================
🔄 步骤 3/5: 转换为 WAV 格式
============================================================

Input #0, s16le, from 'test_48k.pcm':
  Duration: 00:00:05.00, bitrate: 768 kb/s
  Stream #0:0: Audio: pcm_s16le, 48000 Hz, mono, s16, 768 kb/s
Output #0, wav, to 'test_48k.wav':
  Stream #0:0: Audio: pcm_s16le, 48000 Hz, mono, s16, 768 kb/s
size=     469kB time=00:00:04.99 bitrate= 769.4kbits/s speed=70.2x

✅ 转换完成！
```

**为什么需要转换？**
- Orange Pi 的 ALSA 驱动在使用 WAV 格式录音时存在 bug
- 必须先录制 RAW 格式，再用 ffmpeg 转换为 WAV
- 详见：[音频配置完全指南 - 问题 1](AUDIO_SETUP.md#问题-1使用-wav-格式录音失败音量极低或无声)

### 步骤 4：质量验证

脚本会自动分析音频质量：

```python
# 检查音频信号强度
import wave
import numpy as np

with wave.open('test_48k.wav', 'rb') as wf:
    audio_data = wf.readframes(wf.getnframes())
    samples = np.frombuffer(audio_data, dtype=np.int16)
    max_val = np.max(np.abs(samples))
    mean_val = np.mean(np.abs(samples))
```

**输出示例：**
```
============================================================
📊 步骤 4/5: 验证音频质量
============================================================

  音频最大值: 2027
  音频平均值: 156.32
  样本数量: 240000
  时长: 5.00 秒

✅ 音频音量正常！
```

**质量判断标准：**

| 最大值范围 | 质量评估 | 说明 |
|-----------|---------|------|
| < 100 | ❌ 异常低 | 录音失败，可能是设备问题 |
| 100 - 1000 | ⚠️ 较低 | 建议降低 Capture 值（如改为 5） |
| > 1000 | ✅ 正常 | 音频质量良好 |

## 🐛 故障排查

### 问题 1：录音无声或音量异常低

**症状：** 音频最大值 < 100，或播放时听不到声音

**可能原因：**
1. 麦克风未正常工作
2. Capture 音量设置过高（记住：值越小信号越强！）
3. 录音设备错误

**解决方案：**

```bash
# 1. 检查麦克风是否正常
arecord -l

# 2. 降低 Capture 值（增强信号）
amixer set Capture 5   # 或者 1（最强）

# 3. 重新运行测试
./scripts/test_audio_48k.sh
```

### 问题 2：播放无声音

**症状：** 录音正常，但播放时耳机无声音

**可能原因：**
1. 耳机未插好
2. Playback 音量过低
3. Deviceid 未设置

**解决方案：**

```bash
# 1. 检查耳机连接
# 确保耳机插入 3.5mm 接口

# 2. 设置播放参数
amixer set Playback 10
amixer set Deviceid 2

# 3. 测试播放官方音频
aplay -Dhw:ascend310b /opt/opi_test/audio/tianlu.wav

# 4. 如果官方音频能播放，重新运行测试
./scripts/test_audio_48k.sh
```

### 问题 3：录音有"魔音"效果（失真）

**症状：** 播放时声音失真，像"魔音"一样

**可能原因：**
1. 采样率不匹配（使用了 16000 Hz 而不是 48000 Hz）
2. 使用了 WAV 格式录音（触发 Orange Pi bug）

**解决方案：**

```bash
# 确认脚本使用的是 48000 Hz 和 RAW 格式
grep "48000" scripts/test_audio_48k.sh
grep "raw" scripts/test_audio_48k.sh

# 如果脚本正确，问题可能在硬件
# 尝试重启音频服务
sudo systemctl restart alsa-restore
```

### 问题 4：设备忙碌（Device or resource busy）

**症状：** 提示 "Device or resource busy"

**可能原因：**
1. 其他进程正在使用音频设备
2. 之前的测试未正常退出

**解决方案：**

```bash
# 1. 查找占用音频设备的进程
lsof /dev/snd/*

# 2. 杀死占用进程
kill -9 <PID>

# 3. 重新运行测试
./scripts/test_audio_48k.sh
```

## 🔧 脚本定制

### 修改录音时长

编辑脚本，找到 `-d 5` 参数：

```bash
# 原始（5 秒）
arecord -D plughw:0,1 -f S16_LE -r 48000 -c 1 -t raw -d 5 test_48k.pcm

# 修改为 10 秒
arecord -D plughw:0,1 -f S16_LE -r 48000 -c 1 -t raw -d 10 test_48k.pcm
```

### 修改 Capture 音量

编辑脚本，找到 `amixer set Capture 10`：

```bash
# 原始（音量 10）
amixer set Capture 10

# 修改为音量 5（信号更强）
amixer set Capture 5

# 修改为音量 1（信号最强）
amixer set Capture 1
```

### 修改输出文件名

编辑脚本，找到文件名定义：

```bash
# 原始
PCM_FILE="test_48k.pcm"
WAV_FILE="test_48k.wav"

# 修改为自定义名称
PCM_FILE="my_recording.pcm"
WAV_FILE="my_recording.wav"
```
