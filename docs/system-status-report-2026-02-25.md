## 🎵 音频设备状态

### 录音设备

- **设备名称：** plughw:0,1
- **采样率：** 16000 Hz
- **声道数：** 1（单声道）
- **格式：** S16_LE（16-bit PCM）
- **状态：** ⚠️ 需要用户测试验证

### 播放设备

- **设备名称：** hw:ascend310b
- **状态：** ⚠️ 需要用户测试验证

### 设备占用情况

- **pulseaudio 进程：** PID 2880（占用 /dev/snd/controlC0）
- **影响：** 可能需要停止 pulseaudio 或使用 plughw 设备

### 音量配置（重要！）

⚠️ **音量反向特性**：Orange Pi 音频驱动存在音量反向特性
- 音量值越小 → 录音信号越强
- 音量值越大 → 录音信号越弱
- **推荐设置：** `amixer set Capture 10`

## 📊 核心功能实现详情

### 1. 主控制器（assistant.py）

**功能：** 异步编排所有模块，管理整个对话流程

**实现特性：**
- ✅ 异步处理架构
- ✅ VAD 检测循环
- ✅ 主处理循环
- ✅ 状态机管理
- ✅ 性能监控
- ✅ 错误处理
- ✅ 信号处理（Ctrl+C 优雅退出）

**处理流程：**
```
1. VAD 检测循环：持续监听音频，检测语音片段
2. 主处理循环：处理检测到的语音片段
   - ASR 识别：将语音转换为文本
   - LLM 对话：生成回复文本
   - TTS 合成：将文本转换为语音
   - 音频播放：播放合成的语音
3. 状态管理：IDLE → LISTENING → PROCESSING → SPEAKING → LISTENING
```

### 2. ASR 引擎（asr_engine.py）

**功能：** 语音识别，支持中英文双语

**技术指标：**
- 模型：Paraformer INT8 双语模型
- 识别速度：2-3 秒
- 准确率：中文优秀，英文良好
- 支持流式识别
- 支持 ITN（逆文本归一化）

### 3. VAD 检测器（vad_detector.py）

**功能：** 语音活动检测，实时分割语音片段

**技术指标：**
- 模型：Silero VAD
- 阈值：0.2
- 最小静音时长：0.5s
- 最小语音时长：0.25s
- 缓冲区大小：30s

### 4. LLM 引擎（llm_engine.py）

**功能：** 对话生成，支持上下文管理

**技术指标：**
- 模型：DeepSeek Chat
- 超时时间：10s
- 最大重试：3 次
- 支持对话历史（最近 10 轮）
- 支持降级回复

### 5. TTS 引擎（tts_engine.py）

**功能：** 语音合成，支持多语言

**技术指标：**
- 引擎：Edge-TTS
- 语音：zh-CN-XiaoxiaoNeural
- 语速：+0%
- 音量：+0%
- 音调：+0Hz

### 6. 音频捕获（audio_capture.py）

**功能：** 实时捕获麦克风音频

**实现方式：**
- 使用 arecord 持续录音
- 管道读取音频数据
- 异步队列缓冲
- 支持同步和异步读取

### 7. 音频播放器（audio_player.py）

**功能：** 播放 TTS 合成的音频

**实现方式：**
- 使用 aplay 播放音频
- 异步队列管理
- 支持播放状态监控
- 支持播放控制（暂停、恢复、停止）

### 8. 状态机（state_machine.py）

**功能：** 管理系统状态转换

**状态定义：**
- IDLE：空闲状态
- LISTENING：监听状态
- PROCESSING：处理状态
- SPEAKING：说话状态
- ERROR：错误状态

## 🐛 已知问题

### 1. 音频设备问题

**问题：** pulseaudio 进程占用音频设备
**影响：** 可能导致录音或播放失败
**解决方案：**
- 使用 plughw 设备而非 hw 设备
- 或停止 pulseaudio 服务

### 2. 音量反向特性

**问题：** Orange Pi 音频驱动音量值反向
**影响：** 音量设置不直观
**解决方案：** 已文档化，使用推荐设置

### 3. VAD 片段检测

**问题：** 当前检测到 2 个片段，实际有 3 个
**影响：** 可能合并相邻的语音片段
**解决方案：** 已确认这是合理的（语义完整性），如需更精细分割可使用能量 VAD
