## 📊 核心技术选型表

| 子系统 | 功能模块 | 选型技术/服务 | 部署位置 | 选型核心理由 |
|--------|---------|--------------|---------|-------------|
| **听觉** | 语音活动检测 (VAD) | Silero VAD | 本地 (CPU/NPU) | 行业标准，延迟极低 (<30ms)，防止静音上传浪费带宽 |
| **听觉** | 自动语音识别 (ASR) | Sherpa-onnx | 本地 (NPU/CPU) | 支持流式识别，适配昇腾 NPU，响应速度远快于云端 ASR |
| **认知** | 对话逻辑 (LLM) | DeepSeek API | 云端 (API) | 成本极低/免费，推理能力媲美 GPT-4，完全兼容 OpenAI SDK |
| **表达** | 语音合成 (TTS) | Edge-TTS | 云端 (API) | 完全免费，无需 API Key，微软神经网络语音质量极高 |
| **中枢** | 逻辑编排 | Python AsyncIO | 本地 | 处理高并发 I/O（录音、网络请求、播放）的最佳实践 |
| **优化** | TTS缓存 | 本地文件 | 本地 | 缓存常用回复，降低延迟（可选） |
| **监控** | 性能监控 | Python logging | 本地 | 记录各模块延迟，便于优化（可选） |

## 🎯 选型详细说明

### 1. Silero VAD（语音活动检测）

**选型理由：**
- ✅ **延迟极低**：< 30ms，实时性极佳
- ✅ **防止浪费**：过滤静音片段，避免无效上传
- ✅ **轻量级**：CPU占用低，适合嵌入式设备
- ✅ **准确率高**：误触发率低，稳定可靠

**技术参数：**
- 模型大小：~2MB
- 推理延迟：< 30ms
- 准确率：> 95%
- 部署位置：本地 CPU/NPU

---

### 2. Sherpa-ONNX（语音识别）

**选型理由：**
- ✅ **流式识别**：支持实时流式输出，为V4.0做准备
- ✅ **NPU加速**：适配昇腾 310B NPU，充分利用硬件
- ✅ **响应快速**：本地推理 1-2秒，远快于云端 ASR
- ✅ **完全离线**：不依赖网络，隐私保护

**技术参数：**
- 模型：Whisper tiny/base
- 模型大小：40-74MB
- 推理延迟：1-2秒
- 准确率：> 90%
- 部署位置：本地 NPU/CPU

**为什么不用 Faster-Whisper？**
- Sherpa-ONNX 支持流式识别
- 更适合嵌入式设备
- 可以利用 NPU 加速

---

### 3. DeepSeek API（对话逻辑）

**选型理由：**
- ✅ **成本极低**：免费额度充足，成本几乎为零
- ✅ **能力强大**：推理能力媲美 GPT-4
- ✅ **兼容性好**：完全兼容 OpenAI SDK，易于切换
- ✅ **流式支持**：支持流式输出，为V4.0做准备

**技术参数：**
- 模型：deepseek-chat
- 响应延迟：3-5秒
- 支持流式：是
- 部署位置：云端 API

**为什么不用本地 LLM？**
- 本地 LLM 模型太大（> 4GB）
- 推理速度慢，延迟高
- DeepSeek 免费且质量高

---

### 4. Edge-TTS（语音合成）

**选型理由：**
- ✅ **完全免费**：无需 API Key，无使用限制
- ✅ **质量极高**：微软神经网络语音，自然流畅
- ✅ **多种语音**：支持6种中文语音，可自由选择
- ✅ **稳定可靠**：微软官方服务，稳定性有保障

**技术参数：**
- 语音质量：神经网络语音
- 响应延迟：1-2秒
- 支持语音：6种中文 + 多种外语
- 部署位置：云端 API

**为什么不用本地 TTS？**
- 本地 TTS 质量不如云端
- Edge-TTS 完全免费
- 云端 TTS 不是性能瓶颈

---

### 5. Python AsyncIO（逻辑编排）

**选型理由：**
- ✅ **高并发**：非阻塞异步 I/O，高效处理并发
- ✅ **原生支持**：Python 原生库，无需额外依赖
- ✅ **适合场景**：录音、网络请求、播放并行处理
- ✅ **易于维护**：代码清晰，易于理解和维护

**技术参数：**
- 并发模型：异步 I/O
- 性能：高效处理多个 I/O 操作
- 部署位置：本地

---

### 6. 本地文件缓存（TTS缓存）

**选型理由：**
- ✅ **降低延迟**：常用回复直接播放，无需重新生成
- ✅ **节省带宽**：减少云端 API 调用
- ✅ **简单实用**：使用文件系统，无需额外服务

**技术参数：**
- 缓存策略：LRU（最近最少使用）
- 缓存大小：100MB
- 部署位置：本地

---

### 7. Python logging（性能监控）

**选型理由：**
- ✅ **便于优化**：记录各模块延迟，发现性能瓶颈
- ✅ **原生支持**：Python 原生库
- ✅ **灵活配置**：支持多种日志级别和输出格式

**技术参数：**
- 日志级别：DEBUG/INFO/WARNING/ERROR
- 输出格式：时间戳 + 模块 + 延迟
- 部署位置：本地
