## 📋 问题概述

在测试音频播放功能时，发现播放速度异常快，像开了 3 倍速一样。

**用户反馈：**
> "听到声音了，但是像开了倍速，特别快，听的清一点"

---

## 🔍 问题分析

### 症状

- ✅ 能听到声音（说明播放设备配置正确）
- ❌ 播放速度快 3 倍（像开了倍速）
- ❌ 内容难以听清

### 根本原因

**采样率不匹配！**

- **TTS 生成：** 16000 Hz
- **播放设备：** 48000 Hz
- **结果：** 播放器按 48000 Hz 播放 16000 Hz 的音频，速度快了 3 倍

**为什么快 3 倍？**
```
48000 Hz / 16000 Hz = 3
```

播放器认为音频是 48000 Hz，但实际只有 16000 Hz，所以播放速度快了 3 倍。

---

## 🛠️ 解决方案

### 核心思路

**TTS 生成 48000 Hz 音频，与播放设备匹配**

```
TTS 生成（48000 Hz）→ 播放设备（48000 Hz）→ 速度正常
```

### 修复步骤

#### 修改 tts_engine.py

**文件位置：** `realtime_assistant/tts_engine.py`

**修改内容：**

```python
# 修复前（第 151 行）
"-ar", "16000",      # 采样率 16kHz

# 修复后
"-ar", "48000",      # 采样率 48kHz（Orange Pi 硬件原生采样率）
```

**完整代码：**
```python
async def _convert_to_wav(self, input_file: str, output_file: str):
    """
    转换为单声道 WAV

    参数:
        input_file: 输入文件（MP3）
        output_file: 输出文件（WAV）
    """
    cmd = [
        "ffmpeg",
        "-i", input_file,
        "-ar", "48000",      # 采样率 48kHz（Orange Pi 硬件原生采样率）
        "-ac", "1",          # 单声道
        "-y",                # 覆盖输出文件
        output_file
    ]

    process = await asyncio.create_subprocess_exec(
        *cmd,
        stdout=asyncio.subprocess.PIPE,
        stderr=asyncio.subprocess.PIPE
    )

    stdout, stderr = await process.communicate()

    if process.returncode != 0:
        self.logger.error(f"❌ 音频转换失败: {stderr.decode()}")
    else:
        self.logger.debug(f"✅ 音频转换完成: {output_file}")
```

## 🔬 技术细节

### 采样率匹配原理

**为什么必须匹配？**

播放设备按照指定的采样率播放音频：
- 如果音频采样率 < 设备采样率 → 播放速度变快
- 如果音频采样率 > 设备采样率 → 播放速度变慢
- 如果音频采样率 = 设备采样率 → 播放速度正常

**示例：**
```
音频：16000 Hz
设备：48000 Hz
结果：播放速度 = 48000 / 16000 = 3 倍速

音频：48000 Hz
设备：48000 Hz
结果：播放速度 = 48000 / 48000 = 1 倍速（正常）
```

### Orange Pi 音频特性

**硬件原生采样率：** 48000 Hz

**最佳实践：**
- ✅ **录音：** 48000 Hz → 降采样到 16000 Hz（ASR 需要）
- ✅ **播放：** 48000 Hz（直接播放，不转换）
- ✅ **TTS：** 生成 48000 Hz 音频

### 为什么不在播放时转换？

**方案 1：TTS 生成 16000 Hz，播放时转换到 48000 Hz**
- ❌ 需要实时转换，增加延迟
- ❌ 代码复杂
- ❌ 可能引入音质损失

**方案 2：TTS 直接生成 48000 Hz（本次采用）**
- ✅ 无需实时转换，无延迟
- ✅ 代码简单
- ✅ 音质最佳
