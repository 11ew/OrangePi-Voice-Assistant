## 📋 概述

本脚本是一个完整的 ASR 识别测试工具，用于验证 Orange Pi AI Pro 的录音和语音识别功能。

### 主要功能

1. **自动设置音频参数** - Capture、Deviceid
2. **录音测试** - 48000 Hz RAW 格式，避免 Orange Pi WAV bug
3. **自动降采样** - 48000 Hz → 16000 Hz（ASR 需要）
4. **语音识别** - 调用 Paraformer 模型进行识别
5. **详细统计** - 显示音频信息和识别结果

### 步骤 3：自动降采样

脚本会自动将 48000 Hz 降采样到 16000 Hz：

```python
# 降采样：48000 Hz -> 16000 Hz（每 3 个样本取 1 个）
audio_data = audio_data[::3]
```

**输出示例：**
```
📊 音频信息:
   - 原始采样率: 48000 Hz
   - 目标采样率: 16000 Hz
   - 样本数: 80000
   - 时长: 5.00 秒
   - 最大值: 0.0618
   - 平均值: 0.0048
```

**为什么需要降采样？**
- Orange Pi 硬件原生采样率是 48000 Hz
- Paraformer ASR 模型需要 16000 Hz
- 直接录制 16000 Hz 会导致失真（"魔音"效果）
- 先录制 48000 Hz，再降采样到 16000 Hz，音质最佳

## 🐛 故障排查

### 问题 1：识别结果不准确

**症状：** 识别的文本与实际说的话相差很大

**可能原因：**
1. 说话不够清晰
2. 环境噪音太大
3. Capture 音量设置不当

**解决方案：**

```bash
# 1. 检查音频信号强度
# 如果最大值 < 0.01，说明信号太弱

# 2. 降低 Capture 值（增强信号）
amixer set Capture 5   # 或者 1（最强）

# 3. 在安静环境下重新测试
python3 scripts/test_asr_recognition.py

# 4. 说话时靠近麦克风，发音清晰
```

### 问题 2：识别结果为空

**症状：** 显示"❌ 识别失败（结果为空）"

**可能原因：**
1. 录音音量过低
2. 没有说话或说话时间太短
3. ASR 模型加载失败

**解决方案：**

```bash
# 1. 检查音频最大值
# 如果 < 0.001，说明几乎没有声音

# 2. 降低 Capture 值
amixer set Capture 1

# 3. 确保在录音期间持续说话
# 不要只说一个字，要说完整的句子

# 4. 检查 ASR 模型是否存在
ls -lh models/sherpa-onnx-streaming-paraformer-bilingual-zh-en/
```

### 问题 3：录音进程崩溃

**症状：** 提示 "arecord 进程意外结束"

**可能原因：**
1. 音频设备被占用
2. 采样率不支持

**解决方案：**

```bash
# 1. 查找占用音频设备的进程
lsof /dev/snd/*

# 2. 杀死占用进程
kill -9 <PID>

# 3. 重新运行测试
python3 scripts/test_asr_recognition.py
```

### 问题 4：ASR 模型加载失败

**症状：** 提示 "模型目录不存在" 或 "模型文件不存在"

**可能原因：**
1. ASR 模型未下载
2. 配置文件路径错误

**解决方案：**

```bash
# 1. 检查模型是否存在
ls -lh models/sherpa-onnx-streaming-paraformer-bilingual-zh-en/

# 2. 检查配置文件
cat config/asr_config.json

# 3. 如果模型不存在，需要下载
# 参考 docs/phase2-completion-report.md
```

## 🔬 技术细节

### 采样率转换原理

**为什么需要 48000 Hz → 16000 Hz？**

1. **硬件限制**：Orange Pi 音频硬件原生采样率是 48000 Hz
2. **模型要求**：Paraformer ASR 模型训练时使用 16000 Hz
3. **质量保证**：直接录制 16000 Hz 会导致失真

**降采样方法：**
```python
# 简单降采样：每 3 个样本取 1 个
audio_data = audio_data[::3]

# 48000 / 3 = 16000
```

**为什么不用更复杂的降采样算法？**
- 简单降采样已经足够（KISS 原则）
- 性能开销小
- 音质损失可接受

### 音频格式说明

**录音格式：**
- **采样率**：48000 Hz
- **声道数**：1（单声道）
- **位深度**：16 bit
- **格式**：RAW PCM（未压缩）

**为什么使用 RAW 格式？**
- Orange Pi 的 ALSA 驱动在使用 WAV 格式录音时存在 bug
- 必须先录制 RAW 格式，再转换为其他格式
- 详见：[音频配置完全指南 - 问题 1](AUDIO_SETUP.md#问题-1使用-wav-格式录音失败音量极低或无声)
