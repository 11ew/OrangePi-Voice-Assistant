## 研究过程

### 阶段 1：RNNoise 降噪研究

#### 1.1 RNNoise 简介

**RNNoise** 是 Xiph.org 开发的基于 RNN 的音频降噪库：
- 🎯 专门用于语音降噪
- 🚀 轻量级，适合实时处理
- 📦 C 语言实现，性能优秀

#### 1.2 RNNoise 集成

**安装步骤：**
```bash
# 1. 克隆官方仓库
git clone https://github.com/xiph/rnnoise.git

# 2. 编译安装
cd rnnoise
./autogen.sh
./configure
make
sudo make install
sudo ldconfig

# 3. 验证安装
ls -l /usr/local/lib/librnnoise.so*
```

**Python 封装：**
- 创建了 `realtime_assistant/rnnoise_denoiser.py`
- 使用 ctypes 调用 C 库
- 实现了帧级降噪处理

#### 1.3 RNNoise 测试结果

**测试配置：**
- 输入音频：`output/vad_test_volume5.wav` (音量5，最大值 9320)
- 帧大小：480 samples (10ms @ 48kHz)

**降噪效果：**
- ✅ 成功降噪：噪音降低 1.4 dB
- ❌ 信号衰减：信号强度降低 **53%**（最大值从 9320 → 4365）

**VAD 检测结果：**
- 降噪前：检测到 2 个片段
- 降噪后：检测到 **0 个片段** ❌

**结论：**
- ❌ RNNoise 过度降噪，导致信号过弱
- ❌ 反而降低了 VAD 检测效果
- ❌ **不推荐在当前硬件环境下使用**

### 阶段 2：录音音量优化

#### 2.1 音量反向特性发现

**重要发现：** Orange Pi AI Pro 的音频驱动存在**反向特性**！

**测试数据：**

| 音量设置 | 录音信号强度 | 音频最大值 | 文件大小 |
|---------|------------|-----------|---------|
| 1 | ⭐⭐⭐⭐⭐ 最强 | 6380 | 938KB |
| 5 | ⭐⭐⭐⭐ 很强 | 9320 | 938KB |
| 20 | ⭐⭐ 较弱 | ~2000 | 469KB |
| 50 | ⭐ 很弱 | ~500 | 469KB |
| 100 | ☆ 极弱 | ~100 | 469KB |
| 127 | ☆ 几乎无声 | 38 | 469KB |

**结论：**
- ✅ **音量值越小，录音信号越强**
- ✅ **推荐使用音量 1 或 5**
- ❌ 音量值 20 以上会导致信号极弱

#### 2.2 快速录音脚本

创建了 `scripts/quick_record.sh`：
```bash
#!/bin/bash
VOLUME="$1"
OUTPUT_FILE="output/vad_test_volume${VOLUME}"

# 设置音量
amixer set Capture "$VOLUME"

# 录音 10 秒（RAW 格式）
arecord -D plughw:0,1 -f S16_LE -r 48000 -c 1 -t raw -d 10 "${OUTPUT_FILE}.pcm"

# 转换为 WAV
ffmpeg -f s16le -ar 48000 -ac 1 -i "${OUTPUT_FILE}.pcm" -y "${OUTPUT_FILE}.wav"
```

### 阶段 3：能量 VAD 测试

#### 3.1 基于能量的 VAD

**原理：**
- 计算每 100ms 的 RMS 能量
- 使用动态阈值：`threshold = average_energy × 2`
- 根据能量判断语音/静音

**实现：**
- 创建了 `scripts/test_vad_energy.py`
- 支持固定阈值和动态阈值两种模式

#### 3.2 能量 VAD 测试结果

**测试音频：** `output/vad_test_volume1.wav`

**能量分析：**
```
时间轴分析：
0.0-1.2s:  静音（能量 ~190）
1.2-1.6s:  🎤 片段1（能量 1100-1400）
1.6-4.0s:  静音（能量 ~190）
4.0-4.5s:  🎤 片段2（能量 1100-1500）
4.5-7.5s:  静音（能量 ~190）
7.5-9.3s:  🎤 片段3（能量 600-1100）
9.3-10.0s: 静音（能量 ~190）
```

**检测结果：**
- ✅ 成功检测到 **3 个语音片段**！
- ✅ 片段1: 1.2-1.6s (0.4s)
- ✅ 片段2: 4.1-4.5s (0.4s)
- ✅ 片段3: 7.6-9.3s (1.7s)

**结论：**
- ✅ 能量 VAD 对短片段检测更敏感
- ✅ 参数简单直观，容易调优
- ✅ 计算速度快，资源占用低

### 阶段 4：Silero VAD 深度研究

#### 4.1 技术架构分析

**项目使用的技术栈：**
```
EdgeTTSdemo 项目
    ↓ 使用
sherpa-onnx 框架
    ↓ 加载
Silero VAD 模型 (ONNX 格式)
```

**关键发现：**
- 项目使用的是 **sherpa-onnx 框架的 Silero VAD 实现**
- 不是官方 Silero VAD 的 Python 实现
- sherpa-onnx 的实现与官方有差异

#### 4.2 官方 vs sherpa-onnx 对比

| 特性 | 官方 Silero VAD | sherpa-onnx 实现 |
|-----|----------------|-----------------|
| **实现方式** | Python + PyTorch | C++ + ONNX Runtime |
| **参数** | threshold, neg_threshold, min_silence_duration_ms, speech_pad_ms | threshold, min_silence_duration, min_speech_duration, window_size |
| **单位** | 毫秒 (ms) | 秒 (s) |
| **neg_threshold** | ✅ 支持 | ❌ 不支持 |
| **speech_pad_ms** | ✅ 支持 | ❌ 不支持 |

**关键差异：**
1. ❌ 缺少 `neg_threshold`（负阈值）：控制语音结束的判断
2. ❌ 缺少 `speech_pad_ms`（语音填充）：在语音两侧添加填充
3. ⚠️ 后处理逻辑可能不同

#### 4.3 Silero VAD 参数优化

**测试配置：**

| 配置 | threshold | min_silence | 检测片段数 | 结果 |
|-----|-----------|-------------|-----------|------|
| 原始配置 | 0.3 | 1.5s | 2 | ❌ |
| 优化配置 1 | 0.3 | 0.5s | 2 | ❌ |
| 优化配置 2 | 0.25 | 0.5s | 2 | ❌ |
| 优化配置 3 | 0.2 | 0.5s | 2 | ❌ |
| 优化配置 4 | 0.15 | 0.5s | 2 | ❌ |

**结论：**
- ❌ 无论如何调整参数，都只能检测到 2 个片段
- ❌ sherpa-onnx 的实现对短片段和短停顿不够敏感
- ⚠️ 神经网络的"平滑"特性导致片段合并

#### 4.4 为什么 Silero VAD 会合并片段？

**原因分析：**

1. **神经网络的"平滑"特性**
   - Silero VAD 基于 RNN，会考虑上下文信息
   - 对短暂的静音有"记忆"
   - 如果两个片段之间的静音不够长，会认为属于同一段语音

2. **训练数据的影响**
   - 训练时使用的是自然语音对话
   - 自然对话中，短暂停顿（1-2秒）通常不会分割句子
   - 模型学会了"容忍"短暂的停顿

3. **sherpa-onnx 实现的限制**
   - 缺少关键参数（neg_threshold, speech_pad_ms）
   - 后处理逻辑可能不够完善

## 技术发现

### 1. Orange Pi 音频驱动特性

**音量反向特性：**
- ✅ 音量值越小，录音信号越强
- ✅ 推荐使用音量 1 或 5
- ❌ 避免使用音量 20 以上

**RAW 格式录音：**
- ✅ 必须使用 `-t raw` 格式录音
- ❌ 直接使用 `-t wav` 会导致音量极低（驱动 bug）
- ✅ 录音后使用 ffmpeg 转换为 WAV

### 2. sherpa-onnx 框架

**优势：**
- ✅ C++ 实现，性能极高
- ✅ 资源占用低，适合边缘设备
- ✅ 完全离线运行
- ✅ 跨平台支持

**局限性：**
- ❌ VAD 参数不完整（缺少 neg_threshold, speech_pad_ms）
- ❌ 对短片段检测不够敏感
- ⚠️ 与官方 Silero VAD 实现有差异

### 3. VAD 方案对比

| 方案 | 优点 | 缺点 | 适用场景 |
|-----|------|------|---------|
| **sherpa-onnx Silero VAD** | ✅ 抗噪音能力强<br>✅ 语义完整性好<br>✅ 性能优秀 | ❌ 对短片段不敏感<br>❌ 参数不完整 | 自然对话<br>噪音环境 |
| **能量 VAD** | ✅ 对短片段敏感<br>✅ 参数简单直观<br>✅ 计算速度快 | ❌ 对噪音敏感<br>❌ 需要安静环境 | 安静环境<br>精确分割 |
| **RNNoise + VAD** | ✅ 降噪效果好 | ❌ 信号衰减严重<br>❌ 反而降低检测率 | 不推荐 |

## 最终方案

### 选择：继续使用 sherpa-onnx Silero VAD

**决策理由：**

1. **2 个片段也是合理的**
   - 从语义角度看，2 个片段代表 2 个完整的表达
   - 避免过度分割，保持语义完整性
   - 更符合自然语音对话的模式

2. **适合实时语音助手**
   - 每个片段都是完整的表达
   - ASR 识别效果更好
   - 用户体验更流畅

3. **性能和鲁棒性**
   - sherpa-onnx 性能优秀，适合边缘设备
   - 对背景噪音有一定容忍度
   - 完全离线运行，保护隐私

### 备选方案

如果未来需要更精细的分割，可以考虑：

**方案 A：能量 VAD**
- 适合安静环境
- 对短片段检测敏感
- 已验证有效（检测到 3 个片段）

**方案 B：官方 Silero VAD**
- 参数更完整（neg_threshold, speech_pad_ms）
- 可能对短片段更敏感
- 需要安装 PyTorch（依赖较大）

**方案 C：混合 VAD**
- Silero VAD 粗略检测 + 能量 VAD 精细分割
- 结合两种方法的优点
- 实现复杂度较高

```json

  "model": "models/silero_vad.onnx",

  "min_speech_duration": 0.25,
  "window_size": 512,
  "sample_rate": 16000,
  "buffer_size_seconds": 30,
  "energy_threshold": 0.002
```

**优化说明：**

| 参数 | 原始值 | 优化值 | 说明 |

- ✅ 片段2: 2.73s（包含多个短语）
- ✅ 语义完整性好，适合 ASR 识别
