## 系统概述

### 项目背景

EdgeTTSdemo 项目现有的语音对话系统 V2.0 采用**按键触发 + 固定时长录音**的交互模式，虽然功能完整，但交互体验不够自然。实时语音助手系统旨在通过引入 **VAD（语音活动检测）** 和 **异步编排**技术，实现更加自然、流畅的语音交互体验。

### 核心特性

✨ **连续监听** - 无需按键触发，自动检测语音输入
✨ **实时响应** - 异步并行处理，延迟降低 25%
✨ **智能打断** - 支持语音打断，交互更自然
✨ **高准确率** - VAD 误检率 < 5%，ASR 准确率 > 90%
✨ **完全免费** - 所有云端服务（LLM、TTS）完全免费

### 系统对比

| 特性 | 现有系统 V2.0 | 实时助手系统 | 改进 |
|------|--------------|-------------|------|
| **交互方式** | 按键触发 | 连续监听 | 更自然 |
| **录音模式** | 固定时长 | VAD 分割 | 更智能 |
| **编排方式** | Shell 串行 | Python 异步 | 更高效 |
| **总延迟** | ~6s | < 4.5s | 25% ↓ |
| **打断支持** | ❌ | ✅ | 新增 |

## 核心模块

### 1. VAD 检测器 (`vad_detector.py`)

**功能：** 实时检测语音活动，自动分割语音片段

**关键特性：**
- 使用 Silero VAD 模型（sherpa-onnx 内置）
- 双重检测：VAD + 能量检测
- 可配置阈值和时长参数
- 噪音过滤

**配置参数：**
```json
{
  "threshold": 0.5,              // VAD 检测阈值
  "min_silence_duration": 0.5,   // 最小静音时长（秒）
  "min_speech_duration": 0.25,   // 最小语音时长（秒）
  "energy_threshold": 0.01       // 能量阈值
}
```

### 2. ASR 引擎 (`asr_engine.py`)

**功能：** 语音识别，将音频转换为文本

**关键特性：**
- 支持 Paraformer INT8 流式模型（主选）
- 支持 SenseVoice 模型（备选）
- 异步识别接口
- 线程池并行处理
- 支持 NPU 加速

**性能指标：**
- 识别准确率：> 90%
- 识别延迟：< 1.5s
- 支持中英文双语

### 3. LLM 引擎 (`llm_engine.py`)

**功能：** 对话生成，理解用户意图并生成回复

**关键特性：**
- 使用 DeepSeek API（完全免费）
- 异步 HTTP 请求（aiohttp）
- 超时和重试机制
- 降级方案
- 对话历史管理

**配置参数：**
```json
{
  "api_key": "sk-xxx",
  "timeout": 10,                 // 超时时间（秒）
  "max_retries": 3,              // 最大重试次数
  "system_prompt": "你是..."    // 系统提示词
}
```

### 4. TTS 引擎 (`tts_engine.py`)

**功能：** 语音合成，将文本转换为语音

**关键特性：**
- 使用 Edge-TTS（完全免费）
- 异步合成
- 缓存机制
- 支持多种语音
- 自动语言检测

**支持的语音：**
- 中文：晓晓、晓伊、云健、云希、晓萱、云扬
- 英文：Jenny、Aria、Guy 等

### 5. 音频播放器 (`audio_player.py`)

**功能：** 异步音频播放，支持队列和打断

**关键特性：**
- 异步播放队列
- 打断机制
- 使用 aplay 或 sounddevice
- 播放状态管理

### 6. 主控制器 (`assistant.py`)

**功能：** 异步编排所有模块，管理整个对话流程

**关键特性：**
- 异步事件驱动
- 流水线并行处理
- 状态机管理
- 错误处理和恢复
- 性能监控

## 使用指南

### 快速开始

1. **安装依赖**
```bash
# 安装额外依赖
pip install -r requirements_realtime.txt

# 确保已安装 sherpa-onnx
bash scripts/setup_sherpa_onnx.sh
```

2. **下载 VAD 模型**
```bash
# 下载 Silero VAD 模型
wget https://github.com/k2-fsa/sherpa-onnx/releases/download/asr-models/silero_vad.onnx \
  -O models/silero_vad.onnx
```

3. **配置系统**
```bash
# 编辑配置文件
vim config/realtime_config.json
vim config/vad_config.json
```

4. **启动助手**
```bash
# 运行实时语音助手
python3 realtime_assistant_main.py
```

### 配置说明

#### 实时助手配置 (`config/realtime_config.json`)

```json
{
  "vad": {
    "enabled": true,                    // 启用 VAD
    "config_file": "config/vad_config.json"
  },
  "asr": {
    "model_type": "paraformer",         // ASR 模型类型
    "use_streaming": true,              // 使用流式识别
    "num_threads": 4                    // 推理线程数
  },
  "llm": {
    "api_key": "sk-xxx",                // DeepSeek API Key
    "timeout": 10,                      // 超时时间
    "max_retries": 3                    // 最大重试次数
  },
  "system": {
    "log_level": "INFO",                // 日志级别
    "enable_monitoring": true,          // 启用性能监控
    "enable_interrupt": true            // 启用打断机制
  }
}
```

#### VAD 配置 (`config/vad_config.json`)

```json
{
  "threshold": 0.5,                     // VAD 检测阈值 (0-1)
  "min_silence_duration": 0.5,          // 最小静音时长（秒）
  "min_speech_duration": 0.25,          // 最小语音时长（秒）
  "energy_threshold": 0.01              // 能量阈值
}
```

### 常见问题

**Q: VAD 误检测怎么办？**
A: 调整 `threshold` 和 `energy_threshold` 参数，提高检测阈值可以减少误检。

**Q: ASR 识别延迟太高？**
A: 使用 INT8 量化模型，增加 `num_threads` 参数，或启用 NPU 加速。

**Q: LLM 响应超时？**
A: 检查网络连接，增加 `timeout` 参数，或使用降级方案。

**Q: 如何自定义语音？**
A: 修改 `config/realtime_config.json` 中的 `voice_zh` 和 `voice_en` 参数。
