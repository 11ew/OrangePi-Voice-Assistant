## 系统概述

### 项目背景

EdgeTTSdemo 项目现有的语音对话系统 V2.0 采用**按键触发 + 固定时长录音**的交互模式，虽然功能完整，但交互体验不够自然。实时语音助手系统旨在通过引入 **VAD（语音活动检测）** 和 **异步编排**技术，实现更加自然、流畅的语音交互体验。

### 核心特性

✨ **连续监听** - 无需按键触发，自动检测语音输入
✨ **实时响应** - 异步并行处理，延迟降低 25%
✨ **智能打断** - 支持语音打断，交互更自然
✨ **高准确率** - VAD 误检率 < 5%，ASR 准确率 > 90%
✨ **完全免费** - 所有云端服务（LLM、TTS）完全免费

### 系统对比

| 特性 | 现有系统 V2.0 | 实时助手系统 | 改进 |
|------|--------------|-------------|------|
| **交互方式** | 按键触发 | 连续监听 | 更自然 |
| **录音模式** | 固定时长 | VAD 分割 | 更智能 |
| **编排方式** | Shell 串行 | Python 异步 | 更高效 |
| **总延迟** | ~6s | < 4.5s | 25% ↓ |
| **打断支持** | ❌ | ✅ | 新增 |

## 核心模块

### 1. VAD 检测器 (`vad_detector.py`)

**功能：** 实时检测语音活动，自动分割语音片段

**关键特性：**
- 使用 Silero VAD 模型（sherpa-onnx 内置）
- 双重检测：VAD + 能量检测
- 可配置阈值和时长参数
- 噪音过滤

**配置参数：**
```json
{
  "threshold": 0.5,              // VAD 检测阈值
  "min_silence_duration": 0.5,   // 最小静音时长（秒）
  "min_speech_duration": 0.25,   // 最小语音时长（秒）
  "energy_threshold": 0.01       // 能量阈值
}
```

### 2. ASR 引擎 (`asr_engine.py`)

**功能：** 语音识别，将音频转换为文本

**关键特性：**
- 支持 Paraformer INT8 流式模型（主选）
- 支持 SenseVoice 模型（备选）
- 异步识别接口
- 线程池并行处理
- 支持 NPU 加速

**性能指标：**
- 识别准确率：> 90%
- 识别延迟：< 1.5s
- 支持中英文双语

### 3. LLM 引擎 (`llm_engine.py`)

**功能：** 对话生成，理解用户意图并生成回复

**关键特性：**
- 使用 DeepSeek API（完全免费）
- 异步 HTTP 请求（aiohttp）
- 超时和重试机制
- 降级方案
- 对话历史管理

**配置参数：**
```json
{
  "api_key": "sk-xxx",
  "timeout": 10,                 // 超时时间（秒）
  "max_retries": 3,              // 最大重试次数
  "system_prompt": "你是..."    // 系统提示词
}
```

### 4. TTS 引擎 (`tts_engine.py`)

**功能：** 语音合成，将文本转换为语音

**关键特性：**
- 使用 Edge-TTS（完全免费）
- 异步合成
- 缓存机制
- 支持多种语音
- 自动语言检测

**支持的语音：**
- 中文：晓晓、晓伊、云健、云希、晓萱、云扬
- 英文：Jenny、Aria、Guy 等

### 5. 音频播放器 (`audio_player.py`)

**功能：** 异步音频播放，支持队列和打断

**关键特性：**
- 异步播放队列
- 打断机制
- 使用 aplay 或 sounddevice
- 播放状态管理

### 6. 主控制器 (`assistant.py`)

**功能：** 异步编排所有模块，管理整个对话流程

**关键特性：**
- 异步事件驱动
- 流水线并行处理
- 状态机管理
- 错误处理和恢复
- 性能监控

## 使用指南

### 快速开始

#### 1. 安装依赖

```bash
# 安装额外依赖
pip install -r requirements_realtime.txt

# 确保已安装 sherpa-onnx
bash scripts/setup_sherpa_onnx.sh
```

#### 2. 下载 VAD 模型

```bash
# 下载 Silero VAD 模型
wget https://github.com/k2-fsa/sherpa-onnx/releases/download/asr-models/silero_vad.onnx \
  -O models/silero_vad.onnx
```

#### 3. 音频设备配置（⚠️ 重要！）

**Orange Pi AI Pro 音频设备配置：**

在启动实时语音助手前，必须正确配置音频设备。详细配置指南请参考：[音频配置完全指南](AUDIO_SETUP.md)

**快速配置步骤：**

1. **插入耳机**：将 3.5mm 耳机插入开发板的耳机接口

2. **测试音频设备**：
   ```bash
   # 查看录音设备
   arecord -l
   # 应显示：card 0: ascend310b, device 1

   # 查看播放设备
   aplay -l
   # 应显示：card 0: ascend310b, device 0
   ```

3. **设置音量**：
   ```bash
   amixer set Capture 10   # 设置录音音量
   amixer set Playback 10  # 设置播放音量
   amixer set Deviceid 2   # 设置设备ID为2（耳机）
   ```

4. **测试录音和播放**：
   ```bash
   # 录音5秒
   arecord -D plughw:0,1 -f S16_LE -r 16000 -c 1 -t wav -d 5 test.wav

   # 播放测试
   aplay -Dhw:ascend310b test.wav
   ```

**关键配置参数：**
- **录音设备：** `plughw:0,1`（支持采样率自动转换）
- **播放设备：** `hw:ascend310b` 或 `hw:0,0`
- **采样率：** 16000 Hz（ASR 模型要求）
- **声道数：** 1（单声道）

#### 4. 配置系统

编辑 `config/realtime_config.json`，确保音频配置正确：

```json
{
  "audio": {
    "sample_rate": 16000,
    "channels": 1,
    "chunk_size": 1600,
    "device": "plughw:0,1"  // ⚠️ 重要：必须使用 plughw:0,1
  }
}
```

#### 5. 启动助手

```bash
# 运行实时语音助手
python3 realtime_assistant_main.py
```

### 常见问题

**Q: 录音无声音或全是噪音？**
A:
1. 检查音频设备配置是否正确（必须使用 `plughw:0,1`）
2. 检查麦克风音量：`amixer set Capture 10`
3. 测试录音：`arecord -D plughw:0,1 -f S16_LE -r 16000 -c 1 -t wav -d 5 test.wav`
4. 详细排查请参考：[音频配置完全指南](AUDIO_SETUP.md)

**Q: 播放无声音？**
A:
1. 确保耳机已插入 3.5mm 接口
2. 检查播放音量：`amixer set Playback 10`
3. 设置设备ID：`amixer set Deviceid 2`
4. 测试播放：`aplay -Dhw:ascend310b test.wav`

**Q: VAD 误检测怎么办？**
A: 调整 `threshold` 和 `energy_threshold` 参数，提高检测阈值可以减少误检。

**Q: ASR 识别延迟太高？**
A: 使用 INT8 量化模型，增加 `num_threads` 参数，或启用 NPU 加速。

**Q: LLM 响应超时？**
A: 检查网络连接，增加 `timeout` 参数，或使用降级方案。

**Q: 如何自定义语音？**
A: 修改 `config/realtime_config.json` 中的 `voice_zh` 和 `voice_en` 参数。

**Q: 设备忙碌（Device or resource busy）？**
A:
1. 查找占用进程：`lsof /dev/snd/*`
2. 杀死占用进程：`kill -9 <PID>`
3. 或重启 ALSA：`sudo systemctl restart alsa-restore`
