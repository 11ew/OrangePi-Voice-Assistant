**关键问题和解决方案：**

#### 问题 1：使用 WAV 格式录音失败

**问题描述：**
- 使用 `arecord -t wav` 录音时，音频音量极低（最大值只有 1-10）
- 播放录音几乎听不到声音

**根本原因：**
- Orange Pi AI Pro 的 ALSA 驱动在使用 WAV 格式时存在 bug
- 直接使用 `-t wav` 会导致音频数据写入异常

**解决方案：**
```bash
# 步骤 1: 使用 RAW 格式录音
arecord -D plughw:0,1 -f S16_LE -r 48000 -c 1 -t raw -d 5 test.pcm

# 步骤 2: 使用 ffmpeg 转换为 WAV 格式
ffmpeg -f s16le -ar 48000 -ac 1 -i test.pcm -y test.wav
```

**验证结果：**
- RAW 格式录音：音频最大值 3766 ✅ 正常
- WAV 格式录音：音频最大值 1 ❌ 异常

**文档更新：**
- 已将此问题记录到 `docs/AUDIO_SETUP.md`
- 标记为 "⚠️ 问题 1：使用 WAV 格式录音失败（音量极低或无声）"
- 提供了详细的解决方案和验证方法

#### 问题 2：Silero VAD 无法分割语音片段

**问题描述：**
- 用户录制了 3 句话，中间有明显停顿
- Silero VAD 只检测到 1 个连续的语音片段

**根本原因：**
- 录音环境存在持续的底噪（能量值 45-55）
- 底噪填充了语音之间的停顿
- Silero VAD 模型无法区分"轻微的语音"和"底噪"

**解决方案：**
开发了基于能量的 VAD 算法（`scripts/test_vad_energy.py`）：
- 计算每 100ms 的音频能量（RMS）
- 使用能量阈值区分语音和静音
- 通过最小静音时长参数控制分割粒度

**参数配置：**
```python
energy_threshold = 100          # 能量阈值
min_silence_duration = 0.8      # 最小静音时长（秒）
min_speech_duration = 0.3       # 最小语音时长（秒）
```

**验证结果：**
- 成功检测到 3 个语音片段 ✅
- 语音片段时长合理
- 能够有效过滤底噪

## 💡 经验总结

### 技术要点

1. **录音格式必须使用 RAW**
   - Orange Pi AI Pro 的 ALSA 驱动在 WAV 格式下有 bug
   - 必须使用 RAW 格式录音，然后用 ffmpeg 转换
   - 这是一个硬件/驱动层面的问题，无法通过软件规避

2. **VAD 算法需要根据环境选择**
   - Silero VAD：适合安静环境，检测更智能
   - 基于能量的 VAD：适合有底噪环境，更准确
   - 实际应用中可以结合两种算法

3. **底噪是 VAD 的主要挑战**
   - 持续的底噪会填充语音之间的停顿
   - 需要通过能量阈值过滤底噪
   - 或者在更安静的环境下录音

4. **参数调整需要实际测试**
   - VAD 参数（阈值、最小静音时长等）需要根据实际环境调整
   - 不同的录音环境需要不同的参数配置
   - 建议提供多组预设参数供用户选择

### 最佳实践

1. **先测试录音质量**
   - 在进行 VAD 测试前，先确保录音质量正常
   - 使用 RAW 格式录音
   - 检查音频最大值（应 > 1000）

2. **使用能量分析辅助调试**
   - 分析音频的能量变化可以帮助理解 VAD 行为
   - 能量分析可以揭示底噪问题
   - 可以根据能量分析结果调整参数

3. **提供多种 VAD 方案**
   - 不要依赖单一的 VAD 算法
   - 提供多种方案供用户选择
   - 根据实际环境自动选择最佳方案

4. **详细记录问题和解决方案**
   - 硬件相关的问题需要详细记录
   - 提供清晰的解决步骤
   - 帮助后续开发者避免相同问题
