### 音频设备准备

**重要：** 确保音频设备配置正确！

#### 🚀 方法 1：使用一键测试脚本（推荐）

```bash
# 运行自动化测试脚本
./scripts/test_audio_48k.sh
```

该脚本会自动完成：
- ✅ 设置音频参数（Capture、Deviceid、Playback）
- ✅ 录音测试（5秒，48000 Hz RAW 格式）
- ✅ 格式转换（PCM → WAV）
- ✅ 质量验证（检查信号强度）
- ✅ 播放测试（自动播放录音）

**测试检查项：**
- [ ] 录音过程正常（无错误提示）
- [ ] 能听到自己刚才说的话
- [ ] 录音清晰（无失真、无"魔音"效果）
- [ ] 音量合适（不过大或过小）

详细说明请参考：[音频配置完全指南 - 快速测试脚本](AUDIO_SETUP.md#快速测试脚本)

#### 方法 2：手动测试

```bash
# 1. 检查音频设备
arecord -l  # 查看录音设备
aplay -l    # 查看播放设备

# 2. 检查设备占用
lsof /dev/snd/* 2>/dev/null

# 3. 如果有占用，杀死进程
# kill -9 <PID>

# 4. 设置音量（根据 AUDIO_SETUP.md）
amixer set Capture 10
amixer set Playback 10
amixer set Deviceid 2

# 5. 测试录音和播放
arecord -D plughw:0,1 -f S16_LE -r 48000 -c 1 -t raw -d 5 test.pcm
ffmpeg -f s16le -ar 48000 -ac 1 -i test.pcm -y test.wav
aplay -Dhw:ascend310b test.wav
```

**音频设备配置：**
- 录音设备：`plughw:0,1` (48kHz)
- 播放设备：`hw:ascend310b`
- 录音音量：10（音量反向特性：值越小，信号越强）

## 🐛 常见问题和解决方案

### 问题 1：音频设备冲突

**现象：** `Device or resource busy`

**解决方案：**
```bash
# 查找占用进程
lsof /dev/snd/*

# 杀死占用进程
kill -9 <PID>
```

### 问题 2：VAD 不检测语音

**可能原因：**
1. 录音音量太低
2. 麦克风未正确连接
3. VAD 阈值设置不当

**解决方案：**
```bash
# 1. 检查录音音量
amixer | grep Capture

# 2. 测试录音
arecord -D plughw:0,1 -f S16_LE -r 16000 -c 1 -t wav -d 3 test.wav
aplay test.wav

# 3. 调整 VAD 阈值（编辑 config/vad_config.json）
# threshold: 0.2 → 0.15（更敏感）
```

### 问题 3：ASR 识别不准确

**可能原因：**
1. 录音质量差
2. 背景噪音大
3. 说话不清晰

**解决方案：**
1. 调整录音音量
2. 减少背景噪音
3. 说话清晰、语速适中

### 问题 4：LLM 响应慢或失败

**可能原因：**
1. 网络问题
2. API 超时
3. API Key 失效

**解决方案：**
```bash
# 1. 测试网络
ping api.deepseek.com

# 2. 测试 API
python3 scripts/deepseek_chat.py "测试"

# 3. 检查 API Key
grep api_key config/realtime_config.json
```

### 问题 5：TTS 播放无声音

**可能原因：**
1. 播放设备错误
2. 音量设置为 0
3. 耳机未插好

**解决方案：**
```bash
# 1. 检查播放设备
aplay -l

# 2. 测试播放
aplay -Dhw:ascend310b output/phase4_test.wav

# 3. 调整音量
amixer set Playback 10
amixer set Deviceid 2
```
