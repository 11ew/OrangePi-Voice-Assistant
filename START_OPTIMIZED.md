## ⚠️ 常见问题

### 1. arecord 进程意外结束

**现象：**
```
⚠️  arecord 进程意外结束，尝试重启...
✅ arecord 进程已重启
```

**原因：**
- 录音设备被占用
- 音频参数配置问题
- 系统资源不足

**解决方法：**
- 系统会自动重启 arecord 进程
- 如果频繁重启，检查音频设备状态
- 确保没有其他程序占用麦克风

---

### 2. 首字延迟仍然较长

**可能原因：**
1. **网络延迟**
   - 流式处理对网络要求较高
   - VPN 可能增加延迟
   - 建议使用稳定的网络连接

2. **LLM 响应慢**
   - DeepSeek API 可能繁忙
   - 可以考虑使用本地 LLM

3. **句子分割问题**
   - 如果第一句话很长，会等待完整句子
   - 可以调整分割逻辑

**检查方法：**
```bash
# 查看详细日志
python3 realtime_assistant_main.py 2>&1 | tee output.log
```

---

### 3. 播放不流畅

**可能原因：**
1. **TTS 合成慢**
   - Edge-TTS 网络延迟
   - 可以考虑本地 TTS

2. **播放队列阻塞**
   - 检查播放器日志
   - 确认音频设备正常

**解决方法：**
- 启用 TTS 缓存（已默认启用）
- 检查网络连接
- 查看播放器日志

```bash
# 测试 LLM 流式输出
python3 -c "
import asyncio
from realtime_assistant.llm_engine import LLMEngine
from realtime_assistant.utils import load_config

async def test():
    config = load_config('config/realtime_config.json')
    llm = LLMEngine(config['llm'])
    await llm.warmup()

    async for chunk in llm.chat_stream_async('你好'):
        print(chunk, end='', flush=True)
    print()

    await llm.close()

asyncio.run(test())
"
```
